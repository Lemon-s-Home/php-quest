# 15. Разговариваем с объектами

## Запускать всю игру ради одного объекта неудобно

Иногда нужно проверить маленькую вещь:

- создаётся ли меч;
- правильно ли считается урон;
- меняется ли HP после удара;
- что лежит внутри объекта.

Можно каждый раз писать новый `main.php`. Но есть другой способ: открыть PHP-консоль и работать с объектами руками.

Такой инструмент называется REPL.

REPL читает команду, выполняет её, показывает результат и ждёт следующую.

## PsySH

Поставим PsySH:

```bash
composer require --dev psy/psysh
```

Запуск:

```bash
vendor/bin/psysh
```

Если Composer autoload настроен, классы из `src/` доступны прямо в консоли.

## Первый объект в PsySH

Внутри PsySH:

```php
use Game\Weapon\Sword;

$sword = new Sword('Sword', 25);

$sword->describe();
```

Ответ:

```text
=> "Sword (урон: 25)"
```

Теперь можно проверять объекты без отдельного файла.

## Проверяем бойца

```php
use Game\Fighter\Warrior;
use Game\Weapon\Sword;

$hero = new Warrior('Knight', 100, new Sword('Sword', 25));
$orc = new Warrior('Orc', 80, new Sword('Axe', 20));

$hero->describe();
$hero->attack($orc);
$orc->describe();
```

Ты выполняешь бой по одной строке и видишь, что меняется.

Это медленнее, чем обычный запуск программы. Зато лучше видно, где ошибка.

## PsySH и VarDumper

VarDumper тоже работает:

```php
dump($hero);
```

Можно посмотреть объект целиком, потом вызвать метод, потом посмотреть снова.

```php
dump($orc);
$hero->attack($orc);
dump($orc);
```

Так удобно ловить ошибки вроде:

- урон не вычитается;
- HP уходит ниже нуля;
- оружие не то;
- стратегия атаки не меняется.

## Полезные команды

Посмотреть, что есть в объекте:

```php
ls $hero
```

Посмотреть код класса:

```php
show Game\Fighter\Warrior
```

Выйти:

```php
exit
```

Не нужно запоминать всё сразу. На старте достаточно создавать объекты и вызывать методы.

## Шпаргалка

- PsySH — интерактивная PHP-консоль
- установка: `composer require --dev psy/psysh`
- запуск: `vendor/bin/psysh`
- можно создавать объекты руками
- можно вызывать методы по одному
- можно использовать `dump()`
- это инструмент разработчика, поэтому он в `require-dev`

## Напоследок

1. REPL есть во многих языках: Python, Ruby, JavaScript, Clojure. Это один из лучших способов быстро проверить идею.

2. В Laravel есть похожий инструмент `tinker`. Внутри он использует PsySH.

3. PsySH умеет показывать код классов и документацию функций. Это похоже на маленькую лабораторию внутри терминала.

## Практика

### 15.1. Проверь оружие руками

Запусти PsySH.

Создай три оружия:

- `Sword`
- `Axe`
- `Dagger`

У каждого вызови `describe()` и `strike()`.

Задача считается готовой, если ты смог проверить оружие без запуска `main.php`.

### 15.2. Проведи один раунд руками

В PsySH создай героя и монстра.

Проведи один раунд:

```php
$hero->attack($monster);
$monster->attack($hero);
```

После каждого удара вызывай:

```php
dump($hero);
dump($monster);
```

Найди в дампе, сколько HP осталось у каждого.
