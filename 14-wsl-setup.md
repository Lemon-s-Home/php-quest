# 14. Настраиваем WSL: рабочая среда

## Зачем менять рабочее место

До этого момента всё помещалось в один файл. Запустил `php main.php` — и хватит. Дальше начнётся другое.

В следующем уроке появится Composer. Это значит: пакеты, `vendor/`, бинари в `vendor/bin/`, autoload, права доступа, пути. На Windows это работает через раз: то слеши в путях не те, то расширения PHP не подхватились, то Composer не нашёл `php` в `PATH`. Каждая такая мелочь будет съедать время не на ООП, а на возню с системой.

Решение простое: рабочая среда должна быть линуксовой. На Windows для этого есть WSL — Ubuntu, запущенная прямо внутри Windows. Файлы видны из обеих систем, но команды и пути ведут себя по-юниксовому.

## Ставим WSL

В PowerShell от имени администратора:

```powershell
wsl --install -d Ubuntu
```

Перезагрузка. После перезагрузки откроется окно Ubuntu, попросит придумать имя пользователя и пароль. Это пользователь внутри Ubuntu, к Windows-аккаунту он отношения не имеет.

Дальше всё, что написано ниже, выполняется уже внутри Ubuntu.

## Базовые пакеты

Одной командой ставим всё, что нужно для удобной работы:

```bash
sudo apt update
sudo apt install fish git curl jq tree unzip fd-find fzf bat eza \
  php-cli php-mbstring php-xml php-curl php-zip
```

Что это:

- `fish` — shell, в нём приятнее жить, чем в bash;
- `git curl unzip` — без них никуда;
- `jq tree` — общеполезные утилиты;
- `fd-find` — быстрый поиск файлов (`fd`);
- `fzf` — fuzzy-поиск по чему угодно;
- `bat` — `cat` с подсветкой синтаксиса;
- `eza` — современный `ls` с цветами и иконками;
- `php-cli` — сам PHP в терминале;
- `php-mbstring`, `php-xml`, `php-curl`, `php-zip` — расширения, без которых Composer и пакеты будут ругаться.

## Алиасы

В Ubuntu бинари `fd` и `bat` называются иначе, чтобы не конфликтовать со старыми пакетами. Делаем им нормальные имена.

В файле `~/.config/fish/config.fish`:

```fish
alias fd=fdfind
alias bat=batcat
```

Потом перезапусти терминал или выполни:

```fish
source ~/.config/fish/config.fish
```

## Делаем fish shell'ом по умолчанию

```bash
chsh -s (which fish)
```

Закрой терминал, открой заново — теперь приветствует fish.

## Starship

Starship рисует красивый промпт: текущая папка, ветка git, версия PHP, всё в одной строке.

Установка:

```bash
curl -sS https://starship.rs/install.sh | sh
```

Подключение к fish — добавить в `~/.config/fish/config.fish`:

```fish
starship init fish | source
```

Перезапусти терминал.

## Шрифт для иконок

Starship и `eza` рисуют значки из набора Nerd Fonts. Если шрифт обычный, на месте иконок будут квадратики.

Шрифт ставится **в Windows**, не в Ubuntu. Скачай любой Nerd Font (хорошо подходит `FiraCode Nerd Font`), установи как обычный Windows-шрифт. Потом в Windows Terminal:

- открой настройки;
- найди профиль Ubuntu;
- в разделе шрифта выбери `FiraCode Nerd Font` (или какой поставил).

Открой fish заново — иконки на месте.

## Composer

В `apt` Composer устаревший. Ставим официальным способом:

```bash
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer
```

Проверяем:

```bash
php -v
composer --version
```

Обе команды должны напечатать версии без ошибок.

## Клонируем курс

```bash
git clone git@github.com:Lemon-s-Home/php-quest.git
cd php-quest
```

Если SSH-ключ ещё не настроен — пока используй HTTPS:

```bash
git clone https://github.com/Lemon-s-Home/php-quest.git
```

Запусти любой пример:

```bash
php 01-echo.php
```

Если PHP ответил — среда готова.

## Шпаргалка

- WSL — Ubuntu внутри Windows; ставится одной командой `wsl --install -d Ubuntu`
- работаем не в Windows-консоли, а в окне Ubuntu
- shell — fish, промпт — Starship, шрифт — Nerd Font в Windows Terminal
- Composer ставится не из `apt`, а с `getcomposer.org`
- расширения PHP (`mbstring`, `xml`, `curl`, `zip`) нужны Composer'у — без них пакеты не встанут

## Напоследок

1. WSL появился в Windows 10. До этого, чтобы получить нормальный bash, ставили виртуалку или Cygwin. Сейчас Linux запускается одной командой и не отличается от обычного.

2. Fish вместо bash — выбор не для красоты. Fish из коробки делает то, ради чего в bash ставят кучу плагинов: подсказки команд, цветной вывод, нормальную историю.

3. Starship написан на Rust. Он не зависит от shell — один и тот же промпт работает в bash, zsh, fish, PowerShell. Если потом сменишь shell — настройка промпта переедет с тобой.

4. `bat` называется `bat` почти везде, кроме Ubuntu и Debian — там его переименовали в `batcat`, потому что имя `bat` уже занял другой старый пакет. Это редкий случай, когда дистрибутив переименовывает чужой инструмент.

## Практика

### 14.1. Проверь, что среда жива

Открой fish. Должен появиться Starship-промпт с иконками.

Выполни по очереди:

```bash
php -v
composer --version
fd --version
bat --version
eza --version
```

Все пять команд должны напечатать номер версии. Если какая-то ругается — вернись к соответствующей секции.

### 14.2. Открой файл курса через `bat`

Перейди в папку с курсом:

```bash
cd ~/php-quest
```

Открой любой файл урока через `bat`:

```bash
bat 01-php-basics-review.md
```

Текст должен быть с подсветкой синтаксиса и номерами строк.

Запусти один из своих старых файлов:

```bash
php 02-classes.php
```

Если объект создаётся и метод вызывается — среда полностью готова к следующим урокам.
